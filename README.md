□  Make vs Zapier 비교 분석

비교 항목	              Make                                        Zapier
인터페이스	     노드 기반 시각적 캔버스	              단계별 리스트 형태
조건 분기	    Router 1개로 다중 경로 처리	       Filter + Zap 2개 필요
구성 수	     시나리오 1개 / 모듈 6개	               Zap 2개 / Step 8개
Forms               연동직접 연동 가능	                  Sheets 경유 연동
실행 간격	            15분 Polling	                             15분 Polling
오류 추적	     모듈별 상세 로그 확인	               Step별 실행 결과 확인
관리 편의성       한 화면에서 전체 관리	                    Zap별 개별 관리

□  Make vs Zapier 적합성 비교

 ✅ Make를 선택해야 할 때
  - 조건 분기가 2개 이상 필요한 경우
  - 하나의 흐름에서 여러 작업을 동시에 처리할 때
  - 빠른 실행 간격이 필요한 경우 (15분)
  - 복잡한 데이터 가공이 필요한 경우
  - 오류 처리 및 재시도 전략이 필요한 경우

✅ Zapier를 선택해야 할 때
  - 단순한 1:1 자동화 (A 발생 → B 실행)
  - 빠르게 설정하고 바로 사용해야 할 때
  - 자동화 도구를 처음 접하는 경우
  - 실행 간격이 중요하지 않은 업무

→ Make 추천
  이유: 오전/오후 2가지 분기를
        Router 1개로 효율적으로 처리 가능
        Google Forms 직접 연동으로 설정 간편

□ 자동화할 반복 업무 정의 

자동화할 반복 업무 정의
“우리 회사나 팀에서 매일매일, 매주 반복해서 꼭 하는 일 중에 ‘손으로 계속 직접 해야 해서 시간이 많이 들고 실수도 날 수 있는 업무’를 골라서 자동화 할 수 있습니다.

예를 들면,

매일 아침 이메일로 받은 설문 결과를 확인하고 정리하는 일
구글 시트에 수작업으로 데이터 입력하는 작업
특정 조건(예: 완료율, 날짜 등)에 따라 메일을 일일이 보내는 일
여러 앱에서 데이터를 옮기고 분류하는 반복적 업무
정해진 시간에 알림을 보내거나 보고서를 자동으로 발송하는 작업
이런 ‘반복성’ 있고 ‘규칙이 명확한’ 업무들이 자동화하기 딱 좋은 대상이지! 자동화하면 시간도 절약되고, 실수도 줄고, 신나게 다른 창의적인 일에 집중할 수 있습니다.

정리하면,

반복적이고 규칙적인 업무
데이터 입력, 분류, 알림, 메일 발송 등 사람이 반복해서 하는 작업
단순하지만 시간이 많이 드는 일를 자동화 대상으로 잡는 게 가장 좋습니다.


□ 선정 도구 + 선정 이유 

Make를 선택한 이유
1. 복잡한 조건 분기 처리가 너무 편해! 완료율 기준으로 ‘우수’, ‘주의’ 자동 분류 같은 논리도 깔끔하게 구현 가능해.
2. 다양한 데이터 변환과 처리 지원 덕분에, 구글 시트에서 받아온 긴 텍스트(업무 내용, 특이사항)도 가공하기 편해.
3. 전체 흐름을 시각적으로 보여줘서 각 단계가 한눈에 들어와서 설계와 수정이 쉬워!
4. API 호출 같은 고급 기능도 쓸 수 있어서, 업무가 점점 커져도 대응 가능!
5. 단순 자동화보다 한 뼘 더 깊고 꼼꼼한 관리를 원할 때 최적!


□ 워크플로우 설계 문서 

1. 목적과 범위
목적: 직원들이 매일 제출하는 업무 보고서를 자동으로 수집·분류하고, 완료율에 따라 우수/주의 그룹으로 나눠 맞춤형 피드백 메일을 보내는 자동화 시스템 구축
범위:
Google Forms를 통한 보고서 수집
완료율 기준 분기
Google Sheets 내 우수/주의 시트에 데이터 자동 저장
Gmail을 통한 맞춤형 메일 자동 발송
예외 처리 및 오류 모니터링 포함


2. 입력과 출력 데이터 정의
입력 데이터 (Google Forms 필드):

이름 (단답형, 텍스트)
이메일 (단답형, 텍스트)
부서 (단답형, 텍스트)
오늘 완료한 업무 내용 (장문형, 텍스트)
업무 완료율 (숫자, 0~100)
특이사항 (장문형 또는 선택형 텍스트)
출력 데이터:

Google Sheets 우수 직원 시트(이름, 이메일, 부서, 업무 내용, 완료율, 특이사항)
Google Sheets 주의 직원 시트 (같은 필드)
맞춤형 이메일 내용 (칭찬 메일 or 독려 메일)

3. 프로세스 단계별 흐름
직원이 Google Forms에 업무 보고서 제출
Google Forms 응답이 Google Sheets에 자동 기록
Make 시나리오가 Google Sheets 신규 데이터 감지
업무 완료율 기준으로 조건 분기 (≥90% 우수, <90% 주의)
조건별로 각 시트(우수/주의) 해당 행 추가
조건별 맞춤 메일 자동 발송 (Gmail 연동)
작업 성공/실패 로그 기록 및 알림(필요 시)


4. 조건 분기 및 예외 처리
조건 분기:

완료율이 숫자로 올바르게 인식되는지 검증
완료율 ≥ 90% → 우수 처리
완료율 < 90% → 주의 처리
예외 처리:

필수 입력값(이름, 이메일, 완료율) 누락 시 관리자 알림
완료율 값 범위(0~100) 벗어날 경우 오류 로그 기록
메일 발송 실패 시 재시도 또는 관리자 알림 처리
Google Sheets API 오류 발생 시 자동 재시도 설정


5. 책임자 및 실행 환경
책임자:

시스템 관리자 (자동화 시나리오 유지보수, 오류 대응 담당)
부서 관리자 (결과 모니터링 및 피드백 개선)


실행 환경:

Make 자동화 플랫폼
Google Workspace (Forms, Sheets, Gmail) 환경
안정적인 인터넷 연결 및 권한 설정 완료
6. 버전 관리 및 변경 이력
최초 작성일 및 작성자 기록
주요 변경 사항, 변경 일시, 변경 담당자 기록
변경 내용별 간단 요약 기재 (예: 완료율 기준 조정, 메일 템플릿 수정 등)
문서 최신화 주기 또는 변경 발생 시 즉시 업데이트 원칙
